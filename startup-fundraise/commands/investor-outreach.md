# /investor-outreach

> 💡 **커넥터 참조:** 이 커맨드는 [`CONNECTORS.md`](../CONNECTORS.md)에 정의된 도구 플레이스홀더를 사용합니다. 연결된 도구가 많을수록 더 강력해지지만, 웹 검색만으로도 효과적으로 작동합니다.

## 설명

특정 투자자 또는 VC 펀드를 조사하고, thesis 적합도를 평가한 후, 맞춤형 아웃리치를 작성합니다. 웜 인트로 요청(포워딩 가능한 블럽 포함) 또는 콜드 이메일을 자동 생성하며, Day 5/10/21 팔로업 시퀀스도 제공합니다.

**핵심 가치:**
- VC thesis 매칭 (섹터/단계/체크사이즈/지역)으로 적합도 평가
- 포트폴리오 분석 기반 개인화된 훅
- 웜 인트로 vs 콜드 이메일 자동 선택
- 투자자가 기대하는 정보 (트랙션/시장/팀) 강조

## 사용법

```bash
# VC 펀드 파트너에게 아웃리치
/investor-outreach "서울투자파트너스 김철수 파트너"

# 특정 VC 펀드 전체 조사
/investor-outreach "Sequoia Capital Korea"

# 엔젤 투자자
/investor-outreach "이영희 (전 네이버 CPO, 엔젤)"

# AC (액셀러레이터) 프로그램
/investor-outreach "Y Combinator W24 배치"

# 여러 투자자 동시 조사 (배치 모드)
/investor-outreach "프라이머, 블루포인트, 스파크랩스"
```

### 자동 수집 (웹 리서치)

- VC 펀드/파트너 기본 정보
- 투자 thesis & 포트폴리오
- 최근 투자 활동
- 파트너 배경 & SNS

```

   - 이벤트 참석 이력
```

## Thesis 매칭 프레임워크

### 4차원 평가 기준

**1. 섹터 (Sector)**
```
🟢 GREEN: 투자 thesis 핵심 섹터
   예: "B2B SaaS 전문" VC에게 B2B SaaS 기업

🟡 YELLOW: 인접 섹터 또는 확장 관심 영역
   예: "핀테크 전문" VC에게 B2B 결제 SaaS

🔴 RED: Thesis 밖의 섹터
   예: "헬스케어 전문" VC에게 게임 스타트업
```

**2. 단계 (Stage)**
```
🟢 GREEN: 투자 단계 정확히 일치
   예: "Seed 전문" VC에게 Seed 모집 중

🟡 YELLOW: 단계 경계선 (약간 이르거나 늦음)
   예: "Series A" VC에게 Late Seed (A 준비 중)

🔴 RED: 단계 불일치
   예: "Series B+" VC에게 Pre-seed
```

**3. 체크사이즈 (Check Size)**
```
🟢 GREEN: 모집 금액이 VC 체크사이즈 범위 내
   예: "5-30억 투자" VC에게 20억 모집

🟡 YELLOW: 범위 밖이지만 신디케이트 가능
   예: "10-50억 투자" VC에게 5억 모집 (다른 VC와 공동)

🔴 RED: 체크사이즈 현저히 불일치
   예: "최소 100억" VC에게 5억 모집
```

**4. 지역 (Geography)**
```
🟢 GREEN: 핵심 지역
   예: "한국 전문" VC에게 한국 스타트업

🟡 YELLOW: 확장 관심 지역
   예: "한국+동남아" VC에게 동남아 진출 계획 스타트업

🔴 RED: 투자 안 하는 지역
   예: "미국 전용" VC에게 한국 스타트업
```

### 종합 적합도 계산

```
HIGH (최우선 타겟):
  - 4개 차원 모두 GREEN
  - 또는 3개 GREEN + 1개 YELLOW

MEDIUM (고려 가능):
  - 2개 GREEN + 2개 YELLOW
  - 또는 3개 GREEN + 1개 RED (다른 강점으로 커버 가능)

LOW (우선순위 낮음):
  - 2개 이상 RED
  - 또는 4개 모두 YELLOW
```

## 팁

1. **Thesis 적합도가 최우선입니다** — HIGH 적합도 투자자 10명에게 집중하는 것이 MEDIUM 50명보다 효과적입니다.

2. **웜 인트로가 항상 우선입니다** — 콜드 이메일 응답률 1-3% vs 웜 인트로 30-50%.

3. **포워딩 블럽은 짧고 구체적으로** — 연결자가 5초 만에 이해하고 포워딩할 수 있어야 합니다.

4. **콜드 이메일은 100자 이내 핵심 요약** — 투자자는 30초 안에 관심 여부를 결정합니다.

5. **팔로업은 가치 추가** — "메일 보셨나요?"가 아니라 "새로운 트랙션 업데이트"로 접근하세요.

6. **Day 5/10/21 리듬 지키기** — 3회 이상 팔로업하지 마세요. 그 이상은 스팸입니다.

7. **Unit economics를 강조하세요** — 투자자는 트랙션보다 유닛 이코노믹스로 확장 가능성을 판단합니다.

8. **포트폴리오 유사성 언급** — "당신이 이미 성공시킨 모델"이라는 메시지가 강력합니다.

9. **타이밍도 전략입니다** — VC가 펀드 클로징 직후 or 펀드 막바지일 때는 투자 속도가 다릅니다.

10. **배치 모드 활용** — `/investor-outreach "VC1, VC2, VC3"`로 여러 투자자 동시 조사하여 효율을 높이세요.

## 에이전트 병렬 실행

이 커맨드는 실행 시 다음 에이전트를 병렬로 호출합니다:

| 에이전트 | 역할 | 출력 |
|---------|------|------|
| `investor-researcher` | 투자자별 7-10쿼리 리서치 + Thesis 매칭 (GREEN/YELLOW/RED) | 투자자 프로필, 접근경로 |
| `investor-email-writer` | 아웃리치 시나리오 식별 + 맞춤형 이메일 초안 + Day 5/10/21 팔로업 | 이메일 초안 세트 |
| `vc-question-prepper` | (선택) 미팅 성사 후 예상 질문 30개 + 답변 전략 | Q&A 준비 자료 |

**실행 구조:**

```
[/investor-outreach "VC명"] 실행
         │
         ├── investor-researcher ─── 7-10쿼리 리서치 → 프로필 + Thesis 매칭
         │         ↓ GREEN/YELLOW/RED 판정
         └── investor-email-writer ─ 시나리오 식별 → 이메일 초안 + 팔로업 시퀀스

투자자 N명인 경우:
investor-researcher × N  ←  병렬 동시 리서치
```

**투자자 여러 명 배치 실행:**

