# /find-vc

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인하려면 [CONNECTORS.md](../CONNECTORS.md)를 참조하세요.

창업자-facing 투자자 탐색 커맨드입니다. 내부적으로 `/deal-sourcing`의 리서치 방식을 따르되, 사용자는 “딜소싱”이라는 업계 용어를 몰라도 됩니다.

목표는 단순 VC 리스트가 아니라, **누구를 어떤 순서로 만나야 VC funding 성공 확률이 올라가는지**를 정하는 것입니다.

## 사용법

```bash
/find-vc "뭉클랩 AI B2B SaaS Seed Pre-A Korea"
/find-vc "TIPS 가능 AC/VC 찾기"
/find-vc "runway 5개월, 20억 Seed 라운드 리드 후보"
```

## 한국형 확인 축

| 축 | 질문 | 근거 |
|---|---|---|
| 단계 적합도 | Seed/Pre-A에 실제로 투자하는가? | 최근 투자, 포트폴리오, check size |
| 섹터 적합도 | 우리 wedge와 펀드 목적이 맞는가? | thesis, KVIC FundFinder, 조합명 |
| TIPS 가능성 | 운영사이며 추천 가능성이 있는가? | TIPS Partners, 운영사 접수처, 최근 선정팀 |
| 펀드 근거 | 실제 투자 가능한 펀드/조합 근거가 있는가? | KVIC, KVCA DIVA, 공식 공시 |
| 접근 경로 | 콜드보다 강한 소개 경로가 있는가? | 기존 투자자, 고객, 포트폴리오 founder |
| 미팅 순서 | 바로 리드 후보를 만날지, 연습이 필요한가? | objection readiness, IR 완성도 |

## 미팅 단계

| 단계 | 목적 | 예시 |
|---|---|---|
| `PRACTICE` | 반박 수집, 메시지 검증 | 최우선은 아니지만 적합한 VC/AC |
| `VALIDATE` | 투자 가능성, TIPS/펀드 목적, thesis 검증 | 우리 분야/단계가 맞는 후보 |
| `CLOSE` | 리드 투자자 확보 | 정말 받고 싶은 VC/AC |

```markdown

| 항목 | 값 |
|---|---|

| 라운드/목표금액 |  |

| 지역 |  |

## 2. 추천 미팅 순서
| 순서 | 투자자/운영사 | 단계 | 이유 | 공식 근거 | 다음 액션 |

## 3. TIPS / 펀드 근거
```

## 실행 규칙

- 공식 근거와 투자 가능성을 분리합니다. KVIC/KVCA 근거는 “볼 가치”의 증거이지 “투자 확정”이 아닙니다.
- TIPS는 투자자/운영사 관점에서 봅니다. 운영사의 투자심사, 투자 확약, 추천 가능성, R&D 과제 적합성을 함께 확인합니다.
- 이름 있는 VC를 앞에 두지 말고, 연습/검증/클로징 순서로 정렬합니다.
- 근거가 없으면 `needs_evidence`와 필요한 import/search 액션을 명시합니다.

## 관련 루틴

- `/deal-sourcing` — 상세 리서치 내부 루틴
- `/vc-meeting` — 특정 투자자와의 연락/미팅/후속조치
- `/fundraise-pipeline` — 후보 추가 후 pipeline 건강 점검
