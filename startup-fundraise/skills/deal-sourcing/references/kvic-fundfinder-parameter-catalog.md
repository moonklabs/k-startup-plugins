# KVIC FundFinder 파라미터 카탈로그

검증일: 2026-07-04
출처: http://fundfinder.k-vic.co.kr/rsh/rsh/RshMacFnd

KVIC FundFinder는 자연어 검색이 아니라 지정된 분류 코드로 조회한다. 딜소싱 스킬, MCP, CLI는 아래 파라미터 사전을 기준으로 FundFinder snapshot을 import하거나 사용자가 저장한 HTML을 검증한다.

## 요청 구조

| 단계 | HTTP | 경로 | 주요 파라미터 | 설명 |
|---|---|---|---|---|
| 시작 화면 | GET | `/rsh/rsh/RshMacFndInq` | 없음 | 대분류 카드 목록을 반환한다. |
| 펀드 목록 | POST | `/rsh/rsh/RshMacFndLstInq` | `ASCT_CLSS_GRP_CD_FND`, `ASCT_CLSS_CD_FND` | 선택한 대분류/소분류의 펀드 목록을 반환한다. |

### 파라미터

| 파라미터 | 필수 | 예시 | 의미 | 비고 |
|---|---:|---|---|---|
| `ASCT_CLSS_GRP_CD_FND` | 예 | `AA` | FundFinder 대분류 코드 | 첫 화면 카드의 `list('AA')` 값이다. |
| `ASCT_CLSS_CD_FND` | 권장 | `AA02` | FundFinder 소분류 코드 | 목록 화면 좌측 메뉴의 `searchFunds('AA02')` 값이다. 대분류만 보내면 첫 소분류가 기본 선택될 수 있으므로, 정확한 조건 조회에는 소분류까지 저장한다. |

목록 화면에는 운용사 지도 팝업용 hidden field도 포함된다. `OPER_INST_NM`, `HEADOFC_ADDR`, `ADDR`, `REP_TEL_NO`, `HEADOFC_ZIPCD`는 펀드 조건 파라미터가 아니며 기본 DB/리포트에는 저장하지 않거나 마스킹한다.

| 설립 3년 이내 또는 연 매출 20억원 이하 | `AA02` | `EA01` | FundFinder 설명상 초기기업 대상. Seed/Pre-A 기본 후보군이다. |

| 엔젤/개인투자조합 경로 | `AB01` | `AA02` | 초기 투자자/AC 탐색에 보조적으로 본다. |

## MCP/CLI 구현 규칙

- KVIC snapshot 파일 import는 MCP 도구가 아니라 `vc-funds import kvic` CLI로 처리한다. `file_path`를 필수로 받고, `source_url`, `captured_at`, `group_code`, `subcategory_code`를 선택 입력으로 받을 수 있다. 그룹/소분류명은 snapshot row에서 확인될 때 정규화 근거로 보존한다.
- 회사 조건별 펀드 탐색은 별도 `search_funds_for_startup` 도구가 아니라 `search_funds`에 이 카탈로그로 좁힌 조건을 넘기는 방식으로 처리한다.
- 자연어 profile을 바로 POST 파라미터로 만들지 말고, 먼저 이 카탈로그에서 후보 subcategory code 목록을 만든 뒤 snapshot import/검색 근거에 남긴다.
- 동일 스타트업에 여러 코드가 매칭되면 한 번에 단정하지 않고 `primary_codes`, `secondary_codes`, `why_matched`로 반환한다.
- 카탈로그는 화면 snapshot에서 갱신 가능하게 만들되, 자동 갱신 fetcher는 공식 허가 전까지 비활성화한다.
- 결과 리포트에는 `ASCT_CLSS_GRP_CD_FND`, `ASCT_CLSS_CD_FND`를 근거와 함께 남겨 재현 가능하게 한다.
